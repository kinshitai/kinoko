## The Model

A good reference librarian doesn't just search for your keywords. If you ask about "debugging a slow React app," they don't hand you every document containing "React" and "slow." They understand the *problem beneath the question* and find resources that address the actual need — maybe a guide to browser profiling tools or a piece on database query optimization.

## How Injection Works

When an agent session starts, `kinoko run` captures the session context and queries the server for relevant skills:

```
  Agent Session        kinoko run          kinoko serve        SQLite + vec
       │                   │              (Discovery API)           │
       │  session starts   │                    │                   │
       │  (context)        │                    │                   │
       │──────────────────▶│                    │                   │
       │                   │ POST /api/v1/discover                  │
       │                   │───────────────────▶│                   │
       │                   │                    │  embed prompt →   │
       │                   │                    │  cosine similarity│
       │                   │                    │─────────────────▶│
       │                   │                    │  top-K similar   │
       │                   │                    │◀─────────────────│
       │                   │  skills w/ scores  │                   │
       │                   │◀───────────────────│                   │
       │  inject SKILL.md  │                    │                   │
       │◀──────────────────│                    │                   │
       │                   │                    │                   │
```

### The Discovery API

The server endpoint that powers injection:

```
POST /api/v1/discover
{
  "prompt": "session context text",
  "top_k": 5,
  "min_quality": 0.5
}
```

Returns:
```json
{
  "skills": [
    {
      "repo": "default/fix-database-timeout",
      "name": "fix-database-timeout",
      "description": "",
      "score": 0.82,
      "clone_url": "ssh://localhost:23231/default/fix-database-timeout"
    }
  ]
}
```

The server:
1. Embeds the prompt using the ONNX engine (`bge-small-en-v1.5`)
2. Runs cosine similarity search against all stored skill vectors (SQLite vec)
3. Returns skill metadata — the client fetches SKILL.md content via git when needed

### Thresholds and Limits

| Parameter | Default | Purpose |
|-----------|---------|---------|
| `min_quality` | 0.5 | Minimum quality threshold to include a skill |
| `top_k` | 5 | Maximum number of skills returned |
| Input validation | — | Max 50 patterns, 50 library_ids, 2048 embedding dims |

The `min_quality` threshold prevents low-relevance noise. Skills below this threshold are omitted even if they're the "best" matches available — no match is better than a bad match.

### What Gets Injected

Matched skills are formatted as a prompt section and injected into the agent session:

```markdown
## Relevant Knowledge (auto-injected by Kinoko)

The following skills may be relevant to your current task.

### fix-database-timeout (relevance: 0.87)
[Full SKILL.md content]

### go-subprocess-management (relevance: 0.72)
[Full SKILL.md content]
```

The agent sees this as part of its context — it doesn't know or care where the knowledge came from. It just gets better answers.

## Matching Strategy

Injection matches on multiple signals, not just text similarity:

- **Embedding similarity** — semantic closeness between session context and skill content
- **Problem pattern overlap** — skills addressing the same *type* of problem (building, fixing, optimizing)
- **Historical success rate** — skills that have helped in similar situations before

When embeddings are unavailable (server unreachable), injection degrades to pattern-based ranking: `0.7 × PatternOverlap + 0.3 × HistoricalRate`.

## Configuration

Injection is configured through the main `kinoko.yaml` config — there is no separate `injection:` section. The relevant settings are:

- **`server.host` / `server.port`** — where the discover API lives (default `127.0.0.1:23233`)
- **`libraries`** — which skill libraries to search
- **`extraction.ab_test`** — A/B testing for injection effectiveness

The injection client in `kinoko run` automatically connects to the server's discover API using the configured server address.

## The Key Insight

The most useful knowledge often doesn't look like what you asked for. Matching by problem pattern surfaces solutions that keyword search would miss entirely. A skill about debugging Go memory leaks might help someone debugging a Python memory leak — because the *diagnostic approach* transfers, even if the language doesn't.
