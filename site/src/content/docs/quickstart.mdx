## Prerequisites

- Go 1.22+ (`go version`)
- Git (`git --version`)
- SSH client (`ssh -V`)

## 1. Install

```bash
git clone https://github.com/kinshitai/kinoko.git
cd kinoko
go install ./cmd/kinoko
kinoko --version
```

## 2. Initialize Your Workspace

```bash
kinoko init
```

This creates:
- `~/.kinoko/config.yaml` — configuration
- `~/.kinoko/cache/` — local skill cache
- Client SSH key for server authentication

You'll see:
```
🍄 Kinoko initialized successfully!

Your Kinoko workspace is ready at ~/.kinoko/
```

## 3. Set Your Author

Edit `~/.kinoko/config.yaml`:

```yaml
defaults:
  author: "you@example.com"
```

## 4. Start the Server

```bash
kinoko serve
```

On first start, the server downloads the embedding model (`bge-small-en-v1.5`, ~127MB) — this only happens once.

Output:
```
INFO Kinoko infrastructure server started
INFO Embedding engine ready              model=bge-small-en-v1.5 dims=384
INFO Git server ready                    ssh_url=ssh://localhost:23231
INFO Soft Serve HTTP ready               http=:23232
INFO Kinoko API ready                    http=:23233
INFO Hooks enabled                       credential_scan=true indexing=true
```

Three ports open: SSH (23231), Soft Serve HTTP (23232), and the Kinoko API (23233). See [Architecture](/concepts/architecture/#port-layout) for details.

Keep this terminal open. The server self-bootstraps on first run — it creates its data directory and admin keypair automatically.

## 5. Start the Agent Daemon

In another terminal:

```bash
export OPENAI_API_KEY=sk-...
kinoko run
```

Output:
```
INFO Kinoko agent daemon started
INFO Worker pool ready                   workers=4
INFO Scheduler started                   decay_interval=24h
INFO Injection enabled                   skills_loaded=0
```

:::tip[No API key?]
`kinoko run` works without an LLM key in degraded mode — the scheduler and injection still function, only extraction is disabled.
:::

## 6. Test It

In another terminal:

```bash
ssh -p 23231 localhost
```

You should see a Soft Serve TUI or connection info. Press `q` to quit.

You can also check the API:

```bash
curl http://localhost:23232/api/v1/health
```

## What You Have Now

- ✅ Infrastructure server on `ssh://localhost:23231` with API on `:23233`
- ✅ Server-side embedding engine (ONNX, `bge-small-en-v1.5`) — clients stay pure Go
- ✅ Local agent daemon running extraction, injection, and decay
- ✅ Credential scanning active on all pushes
- ✅ Ready for agents to start generating knowledge

## Solo vs Team

What you just set up is the **solo** configuration — all three components on one machine:

| Command | Purpose |
|---------|---------|
| `kinoko init` | Set up your workspace and client config |
| `kinoko serve` | Shared infrastructure (git server, API, hooks) |
| `kinoko run` | Local agent daemon (extraction, injection, decay) |

For **team** use, one machine runs `kinoko serve`, and each team member runs `kinoko init --connect &lt;server-url&gt;` + `kinoko run` locally. See [Architecture](/concepts/architecture/) for details.
