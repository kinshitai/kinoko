## What Kinoko Does

AI agents solve problems all day — debugging breakthroughs, architecture insights, clever workarounds. Every solution dies when the session ends. The next person with the same problem starts from zero.

Kinoko captures what agents learn and delivers it to future sessions. Automatically, securely, without anyone lifting a finger.

```
Agent solves problem → Knowledge extracted → Stored in git → Injected into future sessions
        ↑                                                              |
        └──────────────────────────────────────────────────────────────┘
```

Like mycelium connecting trees in a forest, Kinoko connects agents through shared knowledge — nutrients flowing invisibly through the network, making every node stronger.

## The Pipeline

<CardGrid>
  <Card title="Extraction" icon="seti:pipeline">
    Multi-stage filtering eliminates noise cheaply. Metadata checks first, LLM critique last. Most sessions aren't worth keeping — Kinoko finds the ones that are.

    [How extraction works →](/concepts/extraction/)
  </Card>

  <Card title="Quality Scoring" icon="star">
    7-dimension evaluation, not thumbs up/down. Each skill is scored on specificity, completeness, portability, accuracy, and more. Structured judgment you can debug.

    [How quality works →](/concepts/quality/)
  </Card>

  <Card title="Injection" icon="open-book">
    Pattern matching, not just keyword search. Understands the problem beneath the question. The most useful knowledge often doesn't look like what you asked for.

    [How injection works →](/concepts/injection/)
  </Card>

  <Card title="Decay" icon="warning">
    Knowledge has a half-life. Tactical fixes expire faster than foundational patterns. The system forgets gracefully — and can remember again when evidence warrants it.

  </Card>
</CardGrid>

## Quick Start

```bash
git clone https://github.com/kinshitai/kinoko.git
cd kinoko && go install ./cmd/kinoko
kinoko init && kinoko serve
```

## Project Status

<small>Last updated: 2026-02-16</small>

Kinoko is in active development. What exists today:

- 117 Go files · 456 tests · ~30K lines · 16 internal packages · 73.6% test coverage
- 3-stage extraction pipeline with LLM critic
- 7-dimension quality scoring
- Pattern-based injection with embedding similarity
- Half-life decay with rescue mechanics
- A/B testing framework
- SQLite storage with WAL mode
- Soft Serve git server integration
- Three-command architecture: `init` (setup), `serve` (infrastructure), `run` (agent daemon)
- Git-first design — git is truth, SQLite is derived cache

The system is functional end-to-end. The API surface is stabilizing but not yet frozen.
