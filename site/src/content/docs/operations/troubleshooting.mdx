## Installation

### "command not found: kinoko"

Go's bin directory isn't in your PATH:

```bash
echo 'export PATH=$PATH:$(go env GOPATH)/bin' >> ~/.bashrc
source ~/.bashrc
```

Use `~/.zshrc` for Zsh, `~/.config/fish/config.fish` for Fish.

### "go: command not found"

Install Go from https://golang.org/dl/ and ensure it's in PATH.

## Server Startup

### Port 23231 already in use

```bash
# Find what's using the port
lsof -i :23231        # macOS/Linux
netstat -an | grep 23231  # Linux
```

Fix: stop the conflicting process, or change the port in `~/.kinoko/config.yaml`:

```yaml
server:
  port: 2323
```

### Permission denied creating directories

```bash
chmod 755 ~/.kinoko
mkdir -p ~/.kinoko/data
```

Or change `server.dataDir` in config to a writable location.

### Config validation errors

Kinoko validates config on startup. Common issues:

- **Port out of range:** must be 1–65535
- **Invalid storage driver:** must be `"sqlite"` or `"postgres"`
- **Bad confidence value:** must be 0.0–1.0
- **Library missing path and url:** each library needs one (not both)

## SSH Connection

### Connection refused

1. Is the server running? Start it: `kinoko serve`
2. Check firewall settings
3. Debug with: `ssh -v -p 23231 localhost`

### Authentication issues

```bash
# Check loaded keys
ssh-add -l

# Generate a key if you don't have one
ssh-keygen -t ed25519

# Add to agent
ssh-add ~/.ssh/id_ed25519
```

### Connection hangs

```bash
# Test basic connectivity
nc -v localhost 23231

# Try IP instead of hostname
ssh -p 23231 127.0.0.1

# Clear stale host keys
ssh-keygen -R "[localhost]:23231"
```

## Git / Skills

### "Not a git repository" in skills directory

```bash
cd ~/.kinoko/skills
git init
git add .
git commit -m "Initial commit"
```

### Git user not configured

```bash
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
```

## SKILL.md Validation

### Missing front matter delimiters

```markdown
❌  name: my-skill
    version: 1

✅  ---
    name: my-skill
    version: 1
    ---
```

### Invalid name format

Names must be kebab-case: `my-skill` not `My Skill` or `my_skill`.

### Missing required sections

Body must contain `# Title`, `## When to Use`, and `## Solution`.

### Confidence out of range

Must be 0.0–1.0 (e.g., `0.8`, not `80` or `1.5`).

## Reporting Bugs

Include this info:

```bash
kinoko --version
go version
git --version
uname -a
cat ~/.kinoko/config.yaml
```
