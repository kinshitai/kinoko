# Team Memory

Each team member has a folder. They create small, focused files to preserve knowledge across sessions.

## Structure

```
memory/
  {name}/
    decisions.md      — architectural and design decisions made
    lessons.md        — mistakes, corrections, things learned
    preferences.md    — coding style, tool preferences, patterns favored
    {topic}.md        — any topic-specific knowledge worth preserving
```

## Rules

1. **Small files** — each file covers one topic. Don't let files grow past ~100 lines.
2. **Split when big** — if a file is getting long, split it into focused subtopics.
3. **Date entries** — prefix entries with dates so we know when things were learned.
4. **Read before work** — always read your memory folder before starting a task.
5. **Write after work** — always update relevant files after completing a task.
6. **Be selective** — only persist what's actually useful for future sessions. Not a diary.
