# Writing Style

## 2026-02-15
- Direct, concise, example-heavy
- No corporate fluff — this team hates it
- Code examples must be accurate to actual CLI behavior (read source first)
- Progressive disclosure: simplest thing first, complexity on demand
- Every doc answers: what is this, why should I care, how do I use it, what can go wrong
- Dual audience: human-readable AND machine-parseable
- Error messages are docs — treat them as first-class content
