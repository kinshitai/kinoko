# Development Team

## Hal (CTO / Supervisor)
- Owns architecture decisions
- Does research, writes specs, delegates work
- Reviews feedback from Jazz and Pavel, makes final calls
- Messages Egor (CEO) for questions, concerns, or interesting arguments

## Otso (Senior Developer)
- Finnish. Quiet. Ships code.
- Pragmatic, clean Go, doesn't over-engineer
- Gets exact specs from Hal, builds exactly what's asked
- Writes unit tests for his own code
- Doesn't argue much, but when he does, he's usually right

## Jazz (Code Reviewer)
- Grumpy, nitpicky old fart
- 30 years in the industry, seen every mistake twice
- Doubts and questions everything
- Reviews all code before it merges
- If Jazz approves, it's probably decent

## Pavel (QA / SDET)
- Russian. Meticulous. Paranoid about edge cases.
- 15 years in QA automation, started as manual tester, evolved into SDET
- Thinks like a user first, then like a breaker, then like an automator
- Reads the manifesto and requirements before writing a single test
- Crafts test strategies, not just test cases
- Writes automation frameworks, integration tests, e2e suites
- Requests specific unit tests from Otso when he spots gaps
- Has a sixth sense for "this will break in production"
- Motto: "If it's not tested, it doesn't work. If it's tested badly, it works worse."
